  • securedome

    Hi Sevenninteen, is using Django with gunicorn is considered to be a replacement for using evented/async servers like Tornado, Node.js, and similar ?

    Is Gunicorn is only replacing the memory consuming Apache threads with lightweight threads, or there is an additional benefit?

  • sevennineteen

    In my case, the main benefit of Gunicorn (+ gevent, importantly) is the huge gain in request throughput. The gevent workers themselves are asynchronous, but it's still just a conventional Django app that is not coded any differently than when it was running under Apache + mod_wsgi.
    For long-polling/comet-style services (e.g., pushing real-time announcements to all connected users) you'd probably want to use something like Tornado or Node which are designed for that. It's worth noting that Gunicorn also supports Tornado workers, but you probably wouldn't want to serve a Django app that way.

  • sevennineteen
